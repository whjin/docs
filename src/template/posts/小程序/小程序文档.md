## 生命周期

### 应用维度

- `onLaunch`：小程序初次加载后执行，**只在全局触发一次**
- `onShow`：小程序启动，或者从**后台进入前台时触发**，**监听小程序显示**
- `onHide`：**从前台进入后台时触发**
- `onError`：发生错误时触发
- `onThemeChange`：主题切换时触发
- `onUnhandledRejection``：Promise` 未处理的错误时触发
- 资源占用过高，销毁

### 页面维度

- `onLoad`：**加载页面**时执行
- `onShow`：**显示页面元素**时执行
- `onReady`：页面渲染完成时执行，`render page` 一个页面只会调用一次
- `onHide`：页面隐藏时执行
- `onUnload`：页面卸载时执行
- `wx.redirectTo`
- `wx.navigateBack` `onUnload`

### 组件维度

- `created`：组件实例被创建时执行
- `attached`：组件实例进入页面节点树时执行
- `ready`：组件布局完成时执行
- `moved`：组件实例被移动到页面节点树 anotherTarget 时执行
- `detached`：组件实例被从页面节点树移除时执行
- `error`：组件出错时执行

## 登录流程

- `wx` 登录能力
- `openid`
- `code` `wx.login()` `wx.getUserInfo()`
- 自身的后端，调用微信服务器，获取用户信息
- `wx.request`：`appid`、`appsecret`、`code`
- `openid`：`session_key` `storage`

- 登录态是否过期 `token`
- `wx.checkSession`
- `wx.getSettings` -> `wx.getUserInfo`

## 路由跳转

- `pageModel` `pageStack`
- `wx.navigateTo`：保留当前页面，调整到应用内的页面 -> 新的页面入栈
  - `url` `path?key=value1&key2=value2`
  - `events`
  - `success`
  - `fail`
  - `complete`
- `wx.redirectTo` 重定向，关闭当前页面，跳转到应用内的页面 -> 当前页面出栈，新的页面入栈
- `wx.switchTab`：`tabBar` 关闭所有其他非 `tabBar` 页面 -> 所有页面全部出栈，只留下新的 `tab` 页面
- `wx.navigateBack`：返回上一个页面 -> 页面不断出栈，直到找到目标页面
- `wx.reLaunch`：关闭所有页面，打开到某个指定的页面 -> 页面全部出栈，只留下新的页面
- `wx.getCurrentPages()`

## 发布流程

- 上传代码：`devtools dist` -> `commit message` `version message` + `bundle` + `upload`
- 提交审核：开发管理 - 开发版本 -> 审核提交
- 发布版本：灰度版本 线上版本
- 代码管理服务器拉取分支
- 开发、测试
- 测试后，本地分支 合并到 `master` 分支
- `master` 最新代码，`build` 构建
- 上传
- 审核
- 发布

**`CI/CD`**

自动化 `webHooks`

- 开发提交分支代码
- `master` `Jenkins`
- `compiler`
- `api` `push` 微信后台 审核
